Netatalk 2.4.4

Netatalk 2.4.4 is available!

The Netatalk team is proud to announce the latest version in the Netatalk 2.4 release series.

This release contains bug fixes for the Meson build system.

Notably, Meson now defines the libatalk shared library’s version as “0.0.0” to bring it in line with what Autotools produce. It was previously set to “18” in Meson, which was a mistake when backporting this build system from the 3.x branch.

Additionally, it fixes an issue for Docker users upgrading the 2.4.2 or earlier to 2.4.3. If you had problems starting your Docker image after upgrading, please pull the 2.4.4 image, or later.

All users of the 2.4 release series are recommended to upgrade their versions to 2.4.4.

What’s Changed

  • [2.4] meson: Remove redundant space in psf symlink list by @rdmark in https://github.com/Netatalk/netatalk/pull/1247
  • [2.4] meson: Print spool dir in setup summary by @rdmark in https://github.com/Netatalk/netatalk/pull/1248
  • [2.4] meson: Correct overwrite install logic for package config files by @rdmark in https://github.com/Netatalk/netatalk/pull/1252
  • [2.4] meson: Symlink select binaries and install man page aliases by @rdmark in https://github.com/Netatalk/netatalk/pull/1255
  • [2.4] meson: Adjust libatalk soversion to 0.0.0 by @rdmark in https://github.com/Netatalk/netatalk/pull/1258
  • [2.4] docker: Move config files from the old pkgconfdir location by @rdmark in https://github.com/Netatalk/netatalk/pull/1260

Full Changelog: https://github.com/Netatalk/netatalk/compare/netatalk-2-4-3…netatalk-2-4-4

Note: The Debian deb package distributed with this release is built for Debian 12 Bookworm.
