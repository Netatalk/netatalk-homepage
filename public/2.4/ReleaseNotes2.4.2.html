Netatalk 2.4.2

Netatalk 2.4.2 is available!

The Netatalk team is proud to announce the latest version in the Netatalk 2.4 release series.

When using the Meson build system with the -Dwith-embedded-ssl flag, Netatalk now relies on a system OpenSSL or LibreSSL library as the provider of the CAST cryptography functions. At the same time, we remove source files that were reportedly incompatible with the redistribution terms of this package’s license. All users of the 2.4 release series are strongly recommended to upgrade their versions to 2.4.2.

What’s Changed

  • [2.4] meson: Fix errors in PAM support macro by @dgsga in https://github.com/Netatalk/netatalk/pull/1179
  • [2.4] meson: Fix perl shebang substitution in cnid2_create script by @dgsga in https://github.com/Netatalk/netatalk/pull/1184
  • [2.4] meson: Add option to override system WolfSSL with embedded WolfSSL by @dgsga in https://github.com/Netatalk/netatalk/pull/1175
  • [2.4] Remove obsolete Red Hat Upstart and SuSE SysV init scripts by @dgsga in https://github.com/Netatalk/netatalk/pull/1162
  • [2.4] Use external SSL dependency to provide cast header (#1186) by @dgsga in https://github.com/Netatalk/netatalk/pull/1187
  • [2.4] autotools: gcc 8.5 expects explicit library flags for libgrypt by @rdmark in https://github.com/Netatalk/netatalk/pull/1189
  • [2.4] meson: Fix errors in shadow password macro by @dgsga in https://github.com/Netatalk/netatalk/pull/1191
  • [2.4] Fix Zeroconf on Alpine Linux, and in the Docker configuration by @rdmark in https://github.com/Netatalk/netatalk/pull/1198

Full Changelog: https://github.com/Netatalk/netatalk/compare/netatalk-2-4-1…netatalk-2-4-2

Note: The Debian deb package distributed with this release is built for Debian 12 Bookworm.
