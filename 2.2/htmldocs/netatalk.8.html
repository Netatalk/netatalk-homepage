Name

netatalk — Netatalk AFP server service controller daemon

Synopsis

netatalk

netatalk

Description

netatalk is the service controller daemon responsible for starting and restarting the AFP daemon afpd and the CNID daemon cnid_metad. It is normally started at boot time from /etc/rc.

SIGNALS

SIGTERM

Stop Netatalk service, AFP and CNID daemons

SIGQUIT

Restart AFP and CNID master daemons, but keep all session AFP processes running. Can be used to implement AFP service without downtime.

SIGHUP

Sending a SIGHUP will cause the AFP daemon reload its configuration file.

FILES

afp.conf

configuration file used by afpd and cnid_metad

SEE ALSO

afpd(8), cnid_metad(8), afp.conf(5).
